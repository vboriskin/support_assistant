# Регламент эскалации L1 → L2 / on-call (Интернет-банк)

## Когда L1 ОБЯЗАН эскалировать

1. Затронуты **деньги клиента** — пропавший перевод, дублирующее списание, неверная сумма.
2. **Массовый сбой** — три и более жалобы за час по одному модулю.
3. **Безопасность** — клиент сообщает о компрометации (мошенничество, кража карты, фишинг). Блокировка карты + перевод на службу безопасности немедленно.
4. **Регулятор / суд** — любые запросы от ЦБ / приставов / правоохранительных органов.
5. L1 не нашёл решения **за 20 минут** активной работы.

## Что приложить

- ID клиента (без ФИО — только idx);
- Время начала проблемы;
- Скриншот / лог последних действий (без полного номера карты!);
- Подозрение на конкретный сервис, если есть.

## Безопасность — особый процесс

При подозрении на мошенничество:
1. Заблокировать карту немедленно.
2. Перевести разговор на **fraud-team** (24/7, телефон 900 → 9 → 1).
3. Не разглашать клиенту никаких данных о других транзакциях.

## Эскалация vs передача

- **Эскалация** = L2 берёт тикет в работу, L1 закрывает.
- **Передача** = L1 продолжает быть точкой контакта, L2 даёт совет.

По умолчанию — эскалация. Передача — только если клиент в активном диалоге и нельзя «потерять» разговор.
