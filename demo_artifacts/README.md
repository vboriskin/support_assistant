# Demo-артефакты от поддержки — 3 варианта

Это синтетические данные, имитирующие то, что должна собрать команда поддержки.
Используются, пока реальные артефакты на согласовании.

## Варианты

| Папка | Контекст | Когда брать |
|---|---|---|
| `v1_credit/` | **Кредитный конвейер** (заявки, документы, скоринг, ДКП) | Если ассистент будет на корп-системе оформления кредитов |
| `v2_retail/` | **Интернет-банк для физлиц** (карты, переводы, push, кэшбек) | Если ассистент будет на ДБО физиков |
| `v3_corp/` | **Корпоративный банк** (расчётный счёт, зарплатные проекты, ВЭД) | Если ассистент будет на корпоративном банке |

В каждой папке — 14 артефактов:

```
NN_<name>.<ext>
```

Где NN — номер из таблицы артефактов в UI («Артефакты»). Формат:

- `01_tickets.csv`               — исторические закрытые тикеты (импорт через `/api/ingest/csv`)
- `02_modules_registry.json`     — реестр модулей и категорий
- `03_assignee_groups.json`      — маппинг групп поддержки
- `04_glossary.json`             — глоссарий терминов
- `05_pii_extra.json`            — словарь PII / секретов
- `kb_articles/*.md`             — статьи базы знаний (импорт через UI «КБ → Импорт zip/md»)
- `playbooks/*.md`               — шаблоны ответов клиенту (тоже в KB)
- `08_known_issues.json`         — известные баги
- `eval_cases/*.json`            — эталонные пары вопрос-ответ
- `10_quality_scores.csv`        — исторические оценки качества
- `11_top_categories.csv`        — топ-категорий обращений
- `12_sla_matrix.json`           — SLA-матрица
- `13_escalation_policy.md`      — регламент эскалации
- `14_module_owners.json`        — контакты владельцев модулей

## Как залить в стенд

```bash
# В корне проекта
cd /Users/16728123/Downloads/support_assistant-main/support_assistant
source .venv/bin/activate

# Выбери вариант — например, v1
VARIANT=v1_credit

# 1) Тикеты в индекс
python -m scripts.ingest_tickets demo_artifacts/$VARIANT/01_tickets.csv

# 2) KB и playbook'и через zip — UI → «База знаний» → «Импорт zip/md»
( cd demo_artifacts/$VARIANT && zip -r /tmp/kb.zip kb_articles playbooks )
# В UI выбери /tmp/kb.zip и нажми «Импортировать»

# 3) Eval-кейсы — поштучно через UI «+ В eval-набор» либо скопировать в evals/cases:
mkdir -p evals/cases/demo
cp demo_artifacts/$VARIANT/eval_cases/*.json evals/cases/demo/
```

Остальные артефакты (модули, группы, глоссарий, PII, SLA, контакты, эскалация)
сейчас в проекте не подгружаются автоматически — это **спецификация** того, что вы должны
получить от поддержки. Когда реальные данные приедут, сравните формат с этими образцами.
