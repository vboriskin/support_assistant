# Support Assistant — техническая спецификация

Этот пакет документации описывает целевую систему **RAG-ассистента 1-й линии поддержки** для банковского веб-приложения рассмотрения кредитных заявок. Документация предназначена для реализации с помощью Claude Code или другого AI-инструмента разработки.

## Контекст

Команда 1-й линии поддержки обрабатывает большое количество обращений из разных каналов (почта, мессенджер, чат-бот в веб-приложении, Service Manager). Service Manager работает по сути как сборщик тикетов без процессов вокруг. Цель системы — **снизить нагрузку на 1-ю линию и ускорить time-to-resolution** через:

1. Пайплайн обработки и индексации закрытых тикетов (превращение опыта в поисковый ресурс)
2. RAG-ассистент с гибридным поиском по KB и истории тикетов
3. Автоматическая категоризация входящих обращений
4. Eval-инфраструктура для контроля качества

## Ограничения и принципы

- **Локальная работа.** Всё запускается локально. Зависимости — публичные пакеты Python, заранее зафиксированные версии, в перспективе оффлайн-установка во внутренний контур банка.
- **Чувствительные данные.** В тикетах есть PII (ФИО клиентов, номера заявок, суммы). Маскирование обязательно перед любым LLM-вызовом.
- **LLM — внутренний.** GigaChat через OAuth (внутренний контур Сбера). Адаптер позволяет переключение на YandexGPT или OpenAI-совместимый API.
- **БД — гибкая.** Работает на SQLite (для разработки и MVP без инфраструктуры) и на Postgres с pgvector (для production). Выбор через `.env`.
- **Без Docker для приложения.** Запуск через `python -m uvicorn ...`. Docker может использоваться для опционального запуска Postgres, но не обязателен.

## Стек

| Слой | Технология |
|---|---|
| Бэкенд | Python 3.11+, FastAPI, Pydantic v2 |
| ORM | SQLAlchemy 2.0 async |
| БД | SQLite + sqlite-vec / Postgres + pgvector (опционально) |
| Поиск | Гибридный: векторный (cosine) + полнотекстовый (FTS5/tsvector) |
| Эмбеддинги | sentence-transformers, `intfloat/multilingual-e5-large` |
| NER (PII) | Natasha (русскоязычный) |
| LLM | GigaChat API |
| Frontend | Standalone HTML + vanilla JS + CSS (без сборки, без npm) |
| Тесты | pytest, pytest-asyncio |

## Принципы реализации

1. **Адаптеры для всего внешнего.** LLM, эмбеддинги, vector store, источник тикетов — за интерфейсами. Замена компонента = замена одного модуля.
2. **Все промпты — внешние файлы.** Не в коде. Это позволяет править их без передеплоя и прогонять evals при каждом изменении.
3. **PII-маскирование — обязательный шаг.** До любого LLM-вызова и до индексации.
4. **Логирование всего.** Каждый LLM-вызов: запрос, ответ, latency, токены. Структурное логирование (JSON в prod, человекочитаемое в dev).
5. **Конфигурация через `.env`.** Никаких хардкодов. Pydantic-settings.
6. **Зависимости — фиксированные версии.** Через `pyproject.toml` с lock-файлом (`uv.lock` или `poetry.lock`).
7. **Никаких внешних API без явного разрешения.** Отключить телеметрию во всех библиотеках, которые её собирают.
