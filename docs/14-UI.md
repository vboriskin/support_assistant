# 14. UI

Standalone веб-приложение. Без сборки, без npm, без React. Vanilla JS + HTML + CSS. Отдаётся FastAPI на `/ui`.

## Принципы

1. **Без сборки.** Открывается напрямую с диска или через FastAPI. Не требует node_modules, не требует bundler. Дёшево перенести в банковский контур.
2. **ES Modules.** Импорты — через `import { foo } from './module.js'`. Браузерные импорты, без сборщика. Минимально совместимо с современными браузерами.
3. **Дизайн-токены через CSS custom properties.** Двe темы (светлая и тёмная), переключаются на лету. Префикс — `--app-` (нейтральный).
4. **Никаких сторонних UI-фреймворков.** Все компоненты пишем сами. Это ~10-15 простых компонентов.
5. **Никаких CDN.** Все ресурсы — локальные.
6. **Доступность.** Семантика HTML, focus-states, prefers-reduced-motion, prefers-color-scheme.

## Дизайн-токены

Берём визуальный язык из брендбука внутреннего проекта «Призма» — цветовые палитры, типографику, motion-токены, лоадеры. Но не используем брендинг (название, логотип, концепцию «северной звезды»). Префикс CSS-переменных — нейтральный `--app-` или `--ui-`.

## Страницы

5 страниц, между ними переключение через hash-роутинг (`#/dashboard`, `#/assistant`, …).

### 1. Dashboard (`#/dashboard`)

Главная страница. Сводка системы:

- **Сверху** — карточки KPI: тикетов в индексе, статей KB, прогонов ассистента за неделю, helpful rate.
- **Графики:**
  - Тикеты по модулям (горизонтальный bar chart).
  - Тикеты по статусам (donut).
  - Использование ассистента по дням (line chart).
- **Last activity** — последний прогон ингеста, последний прогон evals.

Использовать `chart.js` (только если можно положить локально) или нативный SVG. Для MVP — нативный SVG, чтобы не тащить зависимости.

### 2. Assistant (`#/assistant`)

Главный экран работы оператора. Двухколоночный layout:

- **Левая колонка (60%):** чат.
  - История сообщений в текущем диалоге.
  - Текстовое поле + кнопка «Спросить» (или Enter).
  - Streaming SSE-ответы, текст появляется по словам.
  - Под ответом — источники (раскрывающийся список), кнопки 👍/👎.
- **Правая колонка (40%):** контекст.
  - Опционально: «прицепить тикет» — выпадающий поиск по тикетам.
  - При прикреплении — данные тикета показываются и используются в `ticket_context`.
- **Сверху:** список диалогов (компактный), переключатель темы.

### 3. Tickets (`#/tickets`)

Список тикетов с фильтрами и поиском.

- Поиск (FTS-запрос).
- Фильтры: модуль, статус, дата.
- Таблица: external_id, тема, статус, модуль, дата.
- Клик по строке → панель с деталями (slide-in справа): тема, описание, выжимка, цитаты, шаги решения.
- Кнопки «Спросить ассистента» (открыть `#/assistant` с прикреплённым тикетом), «Переиндексировать».

### 4. Ingest (`#/ingest`)

Загрузка CSV и мониторинг.

- Drop-zone для CSV.
- Список последних ingest job'ов: статус, дата, прогресс-бар, ссылка на детали.
- В деталях job'а: статистика (total / processed / skipped / failed), распределение по resolution_status, ошибки.

### 5. Evals (`#/evals`)

- Запуск прогона: выбор case_set, кнопка «Запустить».
- История прогонов: таблица с метриками (Recall@5, faithfulness, helpfulness, adversarial pass rate).
- В деталях прогона: пер-кейс результаты, фильтр по «failed cases».

`ui/js/api.js` — обёртка над fetch:

```javascript
const BASE = '/api';

async function request(path, options = {}) {
  const url = `${BASE}${path}`;
  const headers = {
    'Content-Type': 'application/json',
    'X-User-Id': localStorage.getItem('userId') || 'anonymous',
    ...options.headers,
  };
  const resp = await fetch(url, { ...options, headers });
  if (!resp.ok) {
    const err = await resp.json().catch(() => ({ error: 'unknown', message: resp.statusText }));
    throw new ApiError(resp.status, err.error, err.message, err.details);
  }
  return resp.json();
}

class ApiError extends Error {
  constructor(status, code, message, details) {
    super(message);
    this.status = status;
    this.code = code;
    this.details = details;
  }
}

export const api = {
  // Assistant
  async chat({ query, conversationId, ticketContext, filters }) {
    return request('/assistant/chat', {
      method: 'POST',
      body: JSON.stringify({ query, conversation_id: conversationId,
                             ticket_context: ticketContext, filters }),
    });
  },

  // Streaming
  async chatStream({ query, conversationId, ticketContext, onChunk }) {
    const resp = await fetch(`${BASE}/assistant/chat/stream`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ query, conversation_id: conversationId,
                             ticket_context: ticketContext }),
    });
    if (!resp.ok) {
      throw new ApiError(resp.status, 'stream_failed', resp.statusText);
    }
    const reader = resp.body.getReader();
    const decoder = new TextDecoder();
    let buffer = '';
    while (true) {
      const { done, value } = await reader.read();
      if (done) break;
      buffer += decoder.decode(value, { stream: true });
      const lines = buffer.split('\n\n');
      buffer = lines.pop() || '';
      for (const line of lines) {
        const data = line.replace(/^data: /, '').trim();
        if (data === '[DONE]') return;
        try {
          onChunk(JSON.parse(data));
        } catch (e) {
          console.warn('Failed to parse SSE chunk', data);
        }
      }
    }
  },

  // Categorize
  async categorize(req) { return request('/categorize', { method: 'POST', body: JSON.stringify(req) }); },

  // Tickets
  async listTickets(params) {
    const q = new URLSearchParams(params).toString();
    return request(`/tickets?${q}`);
  },
  async getTicket(id) { return request(`/tickets/${id}`); },

  // Ingest
  async listIngestJobs() { return request('/ingest/jobs'); },
  async getIngestJob(id) { return request(`/ingest/jobs/${id}`); },
  async uploadCsv(file) {
    const fd = new FormData();
    fd.append('file', file);
    const resp = await fetch(`${BASE}/ingest/csv`, {
      method: 'POST', body: fd,
      headers: { 'X-User-Id': localStorage.getItem('userId') || 'anonymous' },
    });
    if (!resp.ok) throw new ApiError(resp.status, 'upload_failed', resp.statusText);
    return resp.json();
  },

  // Conversations
  async listConversations() { return request('/conversations'); },
  async createConversation(body) { return request('/conversations', { method: 'POST', body: JSON.stringify(body) }); },
  async getConversation(id) { return request(`/conversations/${id}`); },
  async submitFeedback(conversationId, body) {
    return request(`/conversations/${conversationId}/feedback`,
                   { method: 'POST', body: JSON.stringify(body) });
  },

  // Stats
  async dashboardStats(period = 'week') {
    return request(`/stats/dashboard?period=${period}`);
  },

  // Evals
  async runEvals(req) { return request('/evals/run', { method: 'POST', body: JSON.stringify(req) }); },
  async listEvalRuns() { return request('/evals/runs'); },
  async getEvalRun(id) { return request(`/evals/runs/${id}`); },
};
```

## Theme toggle

`ui/js/components/theme-toggle.js`:

```javascript
const STORAGE_KEY = 'app-theme';

export function initTheme() {
  const stored = localStorage.getItem(STORAGE_KEY);
  if (stored) {
    document.documentElement.setAttribute('data-theme', stored);
  }
}

export function setTheme(theme) {
  if (theme === 'auto') {
    document.documentElement.removeAttribute('data-theme');
    localStorage.removeItem(STORAGE_KEY);
  } else {
    document.documentElement.setAttribute('data-theme', theme);
    localStorage.setItem(STORAGE_KEY, theme);
  }
}

export function currentTheme() {
  const attr = document.documentElement.getAttribute('data-theme');
  if (attr) return attr;
  return window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
}
```

### Toast

```javascript
export function showToast(message, type = 'info') {
  const el = document.createElement('div');
  el.className = `toast toast--${type}`;
  el.textContent = message;
  document.body.appendChild(el);
  requestAnimationFrame(() => el.classList.add('toast--show'));
  setTimeout(() => {
    el.classList.remove('toast--show');
    setTimeout(() => el.remove(), 240);
  }, 3000);
}
```

## Доступность

- Semantic HTML (`<header>`, `<main>`, `<nav>`, `<article>`).
- Focus-visible с явными outline'ами:
  ```css
  *:focus-visible {
    outline: 2px solid var(--app-accent);
    outline-offset: 2px;
  }
  ```
- ARIA-attributes на интерактивных элементах (`aria-current="page"`, `aria-busy="true"`).
- Контраст текст/фон ≥ AA (для обеих тем — проверено токенами).
- Keyboard nav: tab/shift+tab по форме, enter на кнопках, esc на модалках.

## Без сборки

В `<head>` HTML:

```html
<link rel="stylesheet" href="/ui/static/css/theme.css">
<link rel="stylesheet" href="/ui/static/css/base.css">
<link rel="stylesheet" href="/ui/static/css/layout.css">
<link rel="stylesheet" href="/ui/static/css/components.css">

<script type="module" src="/ui/static/js/app.js"></script>
```

`type="module"` критично. Без него браузер не примет import-выражения.

## Чек-лист реализации

1. `theme.css` с обеими темами и переключением по `[data-theme]`.
2. `base.css` с типографикой и сбросом.
3. `layout.css` для сайдбара и основного контента.
4. `components.css` для кнопок, карточек, тостов, загрузчиков.
5. `api.js` с обёрткой fetch и SSE.
6. `app.js` с роутером.
7. Каждая страница — отдельный `pages/<name>.js`, экспортирующий `render(container)`.
8. Theme toggle в сайдбаре, persist в localStorage.
9. Все тексты — на русском (целевая аудитория).
10. Все sizes — через CSS-переменные spacing-scale.
